01
Project Context

What is Veritaserum?

An AI-powered system that assesses the credibility of news articles by verifying individual factual statements — not classifying articles as a whole.

Core pipeline
📰 Article Input
↓
✂️ Claim Extraction (NLP)
↓
🔍 Online Evidence Retrieval
↓
⚖️ Claim Verification + Score
↓
🔴🟡🟢 Highlighted Output + Citations
Key properties
  • Per-claim verification, not article-level classification
  • Confidence scores per statement
  • Links to supporting / contradicting evidence
  • Transparent, explainable reasoning
  • Combines NLP + semantic search + knowledge reasoning

"Online search is the key missing piece — the model must be able to retrieve live evidence, not just rely on parametric memory."

02
Critical Concept

Online Search Is Not Built Into
the Base Model

Common Misconception ✗

"This LLM has online search"

No base LLM intrinsically searches the web. A model's training data has a cutoff date — it cannot retrieve live information on its own. Saying a "model has web search" is imprecise.

Misleading framing
Correct Framing ✓

Reasoning engine + search tool layer

Online search is added as a tool-use / agent layer around a capable reasoning model. The model decides when to search, formulates the query, and synthesises the returned results.

Academically accurate
For your supervisor presentation

Say: "The online search capability in Veritaserum will be implemented as a tool-augmented agent layer wrapped around a capable open-source reasoning model."

03
Model Taxonomy

Two Categories of Relevant Models

Category A

Reasoning Backbone Models

Open-source LLMs that can be connected to a web search tool. They provide the intelligence — claim decomposition, evidence synthesis, verdict generation.

  • Qwen3.5-9B / 27B — most recent (Feb 2026)
  • Qwen3-8B — strong tool-use (May 2025)
  • Qwen2.5-7B-Instruct — practical, lighter
  • Meta-Llama-3.1-8B-Instruct — solid baseline
Category B

Search-Enabled Systems

Pre-built agent systems or HuggingFace Spaces that already demonstrate the search-then-verify workflow. Good reference implementations.

  • OpenResearcher — HF Space, web search + citations
  • SAFE (Google DeepMind) — LLM + Google Search for fact eval
  • Bespoke-MiniCheck-7B — SOTA grounded fact-checking
  • Qwen-Agent — official agent framework with web search tool

Qwen3-8B & Meta-Llama-3.1-8B

May 2025 · Open Source

Qwen3-8B

Leading open-source model in complex agent tasks. Unique thinking/non-thinking mode switch — use thinking mode for fact reasoning, fast mode for simpler claims. 32k native context, 131k with YaRN.

Tool use ✓ Agent tasks ✓ Smaller than 3.5
Jul 2024 · Meta

Meta-Llama-3.1-8B-Instruct

Still a valid reasoning backbone if your supervisor has prior context with Llama. HuggingFace model page explicitly shows tool use with Transformers — web search must be attached as a separate module.

Tool use (external) ✓ No built-in search Familiar baseline

💡 For Veritaserum, prefer Qwen3.5-9B over Llama 3.1 — it's more recent and natively benchmarked for search agent workflows.

Category B — Reference Implementations

Search-Enabled Systems & Spaces

HuggingFace Space

OpenResearcher

Answers questions using live web searches and citations. Closest HF example to Veritaserum's goal: question → web search → grounded answer with links. Excellent reference architecture to study.

Search + Citations ✓
Google DeepMind · 2024

SAFE — Search-Augmented Factuality Evaluator

Breaks a long-form response into atomic claims, sends Google Search queries per claim, determines support. Agrees with human annotators 72% of the time, 20× cheaper. Directly mirrors Veritaserum's per-claim approach.

Academic reference ✓
EMNLP 2024 SOTA

Bespoke-MiniCheck-7B

Current SOTA on the LLM-AggreFact fact-verification benchmark (35 models tested). Grounded fact-checking against source documents. Available on HuggingFace, commercially usable. Ideal as your claim verdict module.

SOTA fact-check ✓
Official · QwenLM

Qwen-Agent Framework

Official agent framework wrapping Qwen3/3.5 models. Includes built-in web search tool, browser automation, RAG, MCP support. The practical bridge between Qwen3.5-9B and online search for Veritaserum.

Plug-and-play search ✓

Supervisor Presentation

What to Say to Your Supervisor

"Online search in Veritaserum is implemented as a tool-augmented layer. We use Qwen3.5-9B (released February 2026, the most recent capable open model on HuggingFace) as the reasoning backbone, connected to a web search tool via the Qwen-Agent framework. For claim verdict generation we use Bespoke-MiniCheck-7B, the current SOTA model on the LLM-AggreFact benchmark."

Reference HF links to show
  • huggingface.co/Qwen/Qwen3.5-9B
  • github.com/QwenLM/Qwen-Agent
  • huggingface.co/bespokelabs/Bespoke-MiniCheck-7B
  • huggingface.co/papers/2403.18802 (SAFE)
Key framing points
  • No base LLM intrinsically searches the web — it's always a tool layer
  • Qwen3.5 is the most recent family — Feb 2026, natively benchmarked on search agents
  • SAFE (Google DeepMind) proves this exact architecture works academically
  • MiniCheck-7B is domain-specific and EMNLP 2024 SOTA — academically defensible
  • The two-module design (backbone + verifier) is cleaner and more maintainable
Bottom line

Qwen3.5-9B + Qwen-Agent (search) + Bespoke-MiniCheck-7B (verdict) is the strongest defensible open-source stack for Veritaserum today.
